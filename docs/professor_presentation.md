# LeBlanc: Automated Prompt Enrichment and Iterative Repair for Secure LLM Code
## Sprint Demo Presentation Outline

**Slide 1: LeBlanc Overview**
- *Title*: LeBlanc
- *Subtitle*: Automated prompt enrichment + iterative repair pipeline for secure LLM code generation
- *Description*: Closing the loop for secure code generation from modern Large Language Models without requiring fine-tuning or accessing model weights.

**Slide 2: The Core Problem**
- Modern models are trained to prioritize functional correctness, not security.
- The average code generated by an LLM contains exploitable security flaws up to ~40% of the time (Pearce et al. 2022).
- Developers ship vulnerable code blindly without static analysis workflows.

**Slide 3: The Three LeBlanc Engines**
- *Engine A*: Prompt Enrichment (CWE-aware keyword injection).
- *Engine B*: Static Analysis Pipeline (Zero-cost native py_compile and robust Semgrep/Bandit checks).
- *Engine C*: Iterative Repair Loop (Auto-feed scanner errors back to the model for up to 3 turns).

**Slide 4: Sprint Improvements - What We Fixed**
- *Speed*: Parallelized LLM calls dropping total scan times from >5 mins to <60s per model block.
- *Architecture*: Eradicated technical debt. Split monolithic scripts into cleanly separated modules.
- *Categorization*: Auto-mapping CWEs into five core vulnerability domains (Injection, Auth, Crypto, File/Path, Deserialization).

**Slide 5: Injection Category End-to-End Metrics**
- *Visual*: Table comparing `llama3-8b` against `llama3-70b` for the full set of Injection scenarios.
- Demonstrates immediate validation of LeBlanc's repair functionality.

**Slide 6: The Research Questions**
- RQ1: Enrichment effect on vulnerability rates.
- RQ2: Model comparison (Safest model out-of-the-box).
- RQ3: Repair effectiveness ceiling.
- RQ4: CWE Difficulty mapping.
- RQ5: The Generational Gap - Contrasting Legacy models and State-of-the-art models.

**Slide 7: Datasets and "Feature Inflation" Methodology**
- Evaluated models using 45 authentic scenario fragments curated from the `SecurityEval` benchmark.
- Shifted away from simple 'security trivia' (which 2026 models easily pass) to **Feature Inflation** scenarios (complex PDF generation, admin dashboards).
- This successfully bypasses native LLM guardrails, allowing Bandit to accurately detect latent organic injections (CWE-89/CWE-78).

**Slide 8: Robust Code Isolation**
- Python-only deep dive: We validate AST via Python's native `compile()` before static execution. Non-python responses correctly trigger "Extraction Failed."
- Locked safe temporary directory usage (`v2/backend/scan_tmp/`) guarantees no crossover state persistence.

**Slide 9: Remaining Work & Edge Cases**

- Full scale analysis execution (running 3 runs each for LLM non-determinism).
